# OTICA_LESSONS_LEARNED_V1

## Objetivo

Registrar as principais descobertas metodológicas obtidas durante a construção do subsegmento Ótica.

Este documento não trata de ótica.

Este documento trata do método utilizado.

---

# LESSON 001

Especialistas não se diferenciam apenas pelo conhecimento técnico.

A principal diferença observada está na redução de recomendações inadequadas.

---

# LESSON 002

Os treinamentos analisados enfatizam:

compreensão antes de recomendação.

A recomendação aparece como consequência da análise.

---

# LESSON 003

Rotina, histórico e expectativa aparecem repetidamente como fatores críticos.

Esses elementos surgiram em:

* treinamentos;
* formação profissional;
* adaptação;
* pós-venda;
* resolução de problemas.

---

# LESSON 004

Grande parte dos problemas observados parece ocorrer quando etapas de entendimento do cliente são ignoradas.

Exemplos:

* rotina ignorada;
* histórico ignorado;
* expectativa ignorada.

---

# LESSON 005

O GPT-4o-mini responde melhor a estruturas determinísticas.

Exemplos:

* sequência de decisão;
* critérios;
* estados;
* objetivos.

Evitar abstrações excessivas.

---

# LESSON 006

Pesquisa e runtime devem permanecer separados.

Pesquisa gera conhecimento.

Runtime utiliza conhecimento destilado.

---

# LESSON 007

Nem todo conteúdo útil para pesquisa é útil para runtime.

A compactação é obrigatória.

---

# LESSON 008

Treinamentos de especialistas são fontes extremamente valiosas.

Em muitos casos são mais úteis do que FAQs ou materiais promocionais.

---

# LESSON 009

Problemas reais de pós-venda ajudam a validar hipóteses construídas durante a pesquisa.

---

# LESSON 010

O método construído para Ótica possui potencial de reutilização em outros segmentos consultivos.

Status:

Validado parcialmente.

Necessita aplicação em novos segmentos.
