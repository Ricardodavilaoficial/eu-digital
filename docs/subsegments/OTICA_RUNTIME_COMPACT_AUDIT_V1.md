# OTICA_RUNTIME_COMPACT_AUDIT_V1

## Objetivo

Auditar o arquivo OTICA_RUNTIME_COMPACT_V1.md antes de qualquer transformação futura em JSON ou Firestore.

## Critérios de auditoria

### 1. Compatibilidade com GPT-4o-mini

O conteúdo deve ser:

- compacto;
- determinístico;
- positivo;
- orientado a ação;
- sem abstrações excessivas.

### 2. Separação correta

O runtime compacto deve conter apenas o que ajuda a resposta.

Não deve conter:

- pesquisa completa;
- justificativas longas;
- citações extensas;
- curso completo;
- conteúdo acadêmico profundo;
- regras específicas de uma ótica assinante.

### 3. Cobertura mínima

O compacto deve cobrir:

- sequência de decisão do consultor;
- expertise técnica básica;
- comportamento do cliente;
- lacunas de informação;
- alertas de risco;
- padrões de confiança;
- falhas de pós-venda;
- campos de personalização do assinante.

### 4. Risco de excesso

Se o conteúdo crescer demais, separar em:

- runtime_short;
- runtime_context;
- runtime_deep_reference.

### 5. Risco de falta

Se a resposta do MEI Robô ficar genérica, verificar se faltam:

- exemplos de aplicação;
- critérios técnicos;
- ações permitidas;
- ações a evitar.
