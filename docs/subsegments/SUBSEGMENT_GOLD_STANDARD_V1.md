# SUBSEGMENT_GOLD_STANDARD_V1

## Objetivo

Definir o padrão oficial de qualidade para subsegmentos do projeto MEI ROBÔ.

Um subsegmento somente poderá ser considerado concluído quando atender aos critérios definidos neste documento.

---

# CRITÉRIO 001

Pesquisa realizada em fontes reais.

Exemplos:

* treinamentos;
* cursos;
* especialistas;
* fabricantes;
* entidades do setor.

---

# CRITÉRIO 002

Existência de modelo canônico.

Arquivo esperado:

SUBSEGMENT_CANONICAL_MODEL

---

# CRITÉRIO 003

Existência de pesquisa estruturada.

Arquivo esperado:

SUBSEGMENT_TRAINING_RESEARCH

---

# CRITÉRIO 004

Existência de runtime compacto.

Arquivo esperado:

SUBSEGMENT_RUNTIME_COMPACT

---

# CRITÉRIO 005

Separação clara entre:

pesquisa

e

runtime

---

# CRITÉRIO 006

Compatibilidade com GPT-4o-mini.

O conteúdo deve privilegiar:

* estados;
* critérios;
* objetivos;
* sequências;
* estruturas determinísticas.

---

# CRITÉRIO 007

Existência de padrões comportamentais.

Exemplos:

* behavioral clusters;
* information gaps;
* risk alerts.

---

# CRITÉRIO 008

Existência de padrões de especialista.

Exemplos:

* reframing;
* trust building;
* specialist vs beginner.

---

# CRITÉRIO 009

Existência de validação.

A pesquisa deve ser confrontada com:

* problemas reais;
* falhas;
* retrabalho;
* pós-venda.

---

# CRITÉRIO 010

Possibilidade de personalização pelo assinante.

O subsegmento deve prever:

* diferenciais;
* produtos;
* marcas;
* políticas;
* processos próprios.

---

# RESULTADO

Quando todos os critérios forem atendidos:

O subsegmento poderá ser considerado referência para futuras implementações.
